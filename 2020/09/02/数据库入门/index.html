    数据库入门基础

    发表于 2020-09-02- 更新于 2020-09-19
    3.4k- 12m-
    数据库

    什么是数据库？

    数据库系统由DB，DBMS，SQL三部分组成的系统化数据管理系统。

    1. DB: 即Data Base 也就是数据库，存放数据的电子仓库
    2. DBMS：Data Base Management System，即数据库管理系统，也可以叫做数据库软件，常见的数据库软件有Mysql，MongoDB，Oracle，SqlServer，本博客主要以学习MariaDB为主。
    3. SQL：结构化查询语言，几乎所有的DBMS都支持SQL，并且简单易学
    4. DBA：数据库管理员，也就是操作数据库的程序猿

    为什么要使用数据库？

    在实际的应用中，我们往往需要持久化保存数据，普通的配置文件或文本文件可以使用txt或者json,yml等文件格式进行保存，但是当数据量达到十万百万的时候，传统的文件保存就显得非常吃力和繁琐了，为了便于对大量数据进行保存，读取等相关操作，我们就需要使用到 数据库

    特点

    1. 单个数据库由多个表组成，类似excel中的sheeet表，而单个表由多个列组成，可以像理解excel中的数据表那样理解数据库中的表和列。
    2. 将数据放到表中
    3. 一个数据库可以有多个表，每个 表名具有唯一性，并且每个数据库名也是唯一的
    4. 表由列组成，我们也称为 字段，所有表都是由一个或多个列组成的

    数据模型

    在将实际的对象存入数据库时，我们往往需要将此对象抽象为数据模型来进行存储。而一个好的数据模型可以更加形象地揭示事物的特征，从而更好地通过数据来帮助人们解决问题。

    数据模型是一组概念，描述或者公式的集合，也就是为一组对象数据设计一个用来处理它的模型。

    举个例子，比如我们要存储一个女朋友对象，那么我们需要为这个女朋友对象设计一个数据模型。模型中应该包括女朋友的姓名、年龄、性别、头发颜色、眼睛颜色、喜好、每月工资等等……我们需要为对应的数据设计最合适的数据类型。比如姓名应该是String类型，而年龄应该是int类型或为了减少空间可以优化为byte类型，再如女朋友的每月工资不固定，需要取一个平均值，这时候就工资可以用平均数来表示：

    等等。这个过程就叫做设计数据模型。

    数据模型一般应该满足：

    1. 数据模型能够真实地模拟实际问题。
    2. 数据模型本身容易理解
    3. 数据模型易于计算机实现

    数据模型组成要素

    1. 数据结构： 数据结构主要描述数据的类型，内容，性质以及数据间的联系等。数据结构是数据模型的基础。数据操作和约束都建立在数据结构上。

    2. 数据操作：数据操作主要是在相应数据结构上进行的操作类型和操作方式,

      常见的数据操作就是对数据库中的数据进行增删查改。

    3. 数据约束： 数据完整性约束是为例确保数据的正确性和相容性，而对输入的数据进行一系列约束行为。比如你的对象的性别只能是男或女，那么在输入的时候就要对输入的数据进行数据约束，确保性别只有男或女。

    数据库系统的体系结构

    1. 单用户数据库

      单用户数据库系统是一个数据库系统只能被一个用户使用，也就是只有一个管理员对该数据库进行管理，仅可以在本地的计算机中使用该用户的账号进行访问。而其它人不能访问。

    2. 主从数据库系统

      故名思意，主从数据库是由一台主机带多个附属终端的多用户数据库系统。

    数据库模型分类

    1. 层次模型

      层次模型就是以“上下级”的层次关系来组织数据的一种方式，层次模型的数据结构看起来就像一棵树

    2. 网状模型

      网状模型把每个数据节点和其它很多节点都连接起来，它的数据结构看起来就像很多城市之间的路网。

    3. 关系模型
      1. 关系模型把数据看作是一个二维表格，任何数据都可以通过行号+列号来唯一确定，它的数据模型看起来像一个excel。
      2. 表的每一行称为记录（Record），记录是一个逻辑意义上的数据。
      3. 表的每一列称为字段（Column），同一个表的每一行纪录都有多个字段。一张表中的的每一天记录都包含若干定义 好的字段。同一个表的所有记录都有相同的字段定义。

    主键

    对于关系表，任意两条记录不能重复，不能重复不是指两条记录不完全相同，而是指 能够通过某个字段唯一区分出不同的记录这个字段被称为主键。+

    主键是用来唯一定位记录的，修改了主键，往往会造成一系列影响，因此 不要随便修改主键。

    主键的选取会对业务开发产生重要影响，因此，选取主键的基本原则：不用任何业务相关的字段作为主键

    一般把主键字段命名为id，常见的id字段类型有：

    1. 自增整数模型：数据库会在插入数据时自动为每一条记录分配一个自增证书，每创建一条记录数据，id就会自增，这样我们就完全不用担心主键重复。
    2. 全局唯一GUID模型： 全局唯一标识符（GUID，Globally Unique Identifier）是一种由算法生成的二进制长度为128位的数字标识符，GUID算法通过网卡MAC地址，时间戳和随机数保证人以计算机在任意时间生成的字符串都是不同的。 使用这种全局唯一的字符串作为主键 ，可以保证主键不重复。
    联合主键

    关系型数据库实际上还允许通过多个字段唯一标识记录，即两个或更多的字段都设置为主键，这种主键被称为联合主键

    没有必要的情况下，我们尽量不使用联合主键，因为它给关系表带来了复杂度的上升。

    小结

    主键得选取不要带有业务含义，应该使用BigInt自增或者GUID

    外键（FOREIGN KEY）

    ​ 在一个表中表达“一对多”的关系，比如一个课程表中需要找出某个记录中某节课字段对应的所有用户数据。这时候就需要外键来实现这种”一对多“，在students表中我们可以加入一个class_id字段，让它的值与class表的id字段对应，这样就可以实现两个表连接起来。

    外键的约束

    拥有外键后我们往往需要对外键进行约束以保证他的数据合法。

    （也可以不进行约束，在代码逻辑层面上进行约束）

    添加约束
    ALTER TABLE students
ADD CONSTRAINT fk_class_id
FOREIGN KEY (class_id)
REFERENCES classes (id);

    这里在students表中添加了一个外键约束规则 fk_class_id

    使用FOREIGN KEY (class_id)指定class_id字段为外键

    REFERENCES classes (id)将class_id与classes表的id进行关联

    也就是说，class_id的值与class表中的id进行绑定同步。

    删除外键约束
    ALTER TABLE students
DROP FOREIGN KEY fk_class_id;

    定位到students表，然后将外键约束fk_class_id进行删除。

    索引（INDEX）

    为了提高查询数据的速度，这时需要使用索引来提升速度。

    索引是关系型数据库中某一列或多个列的值进行 预排序的数据结构。通过索引可以让数据库不必扫描整个表，而是直接定位到符合条件的记录，这样就大大加快了查询速度。

    例子以及注意点

    对于一个students表，创建根据score字段进行查询的索引：

    ALTER TABLE students
ADD INDEX idx_score (score);

    这样就创建了一个名为 idx_score的索引，他指向students表的score字段。

    索引的效率取决于索引列的值是否散列，也就是他的值是不是互相不同的。如果值不同，那么相对索引效率就高，反之则低。一张表可以有多个索引，但是索引越多插入，更新和删除记录的速度就越慢（索引需要随记录而更新）。

    每张表中的主键是唯一的。所以一般可以使用主键作为索引来实现快速查询。

    一个普通索引可能查出多个相同的记录。

    唯一索引（ UNIQUE INDEX ）

    有时我们会遇到某些特殊需求，需要采用唯一索引。比如身份证，用户账号等，这些东西具有业务含义，不能作为主键，但是又不能重复，需要唯一约束。因此这个时候就需要唯一索引或唯一约束。也就是索引列的值只能唯一，一个索引只对应一条记录（可以为空值）

    例子

    唯一索引：

    ALTER TABLE students
ADD UNIQUE INDEX uni_name (name);

    为students表中的name字段添加唯一索引

    唯一约束：

    ALTER TABLE students
ADD CONSTRAINT uni_name UNIQUE (name);

    数据库查询

    1. 条件查询

      使用WHERE+判断语句查询

      SELECT * FROM students
WHERE score >= 80;
//在students表中查询score>=80的记录

      AND语句

      SELECT * FROM students 
WHERE score >= 80 AND gender = 'M';
//在students表中查询score>=80并且Gender数据为M的记录

      OR语句

      SELECT * FROM students
WHERE score >= 80 OR gender = 'M';
//在students表中查询score>=80或者Gender数据为M的记录

      NOT语句

      SELECT * FROM students 
WHERE NOT class_id = 2;
//在students表中查询calss_id！=2的记录
    2. 投影查询

      可以在SELECT后面指定返回的列，来让返回的记录只包含我们所想要的列。

      SELECT id, score, name
FROM students;
//只会查询的students表中的id，score，name三列
    3. 排序

      ORDER BY语句（默认由小到大）

      SELECT id, name, gender, score
FROM students 
ORDER BY score;
#从students表中获取id, name, gender, score四列，将其以score为条件由低到高进行排序

      DESC 倒序语句

      SELECT id, name, gender, score
FROM students 
ORDER BY score DESC;
#后面加了DESC就可以将其由高到低排序
    4. 分页查询

      当一张表数据过多时，我们会采取分页查询，也就是把一堆数据分成好几个部分，一个部分一个部分地看

      LIMIT <页数> OFFSET<记录数>语句

      SELECT *
FROM students
LIMIT 2 OFFSET 0;
#将查询到的记录以每2条作为一页，并且显示第0页
#LIMIT后面加每张表的记录条数，
OFFSET后面加要结果集从第几条记录开始
比如查询第二页，那么OFFSET后面应该设置为3
    5. 聚合查询

      我们需要查询一张表有多少记录的时候就需要用到聚合查询

      COUYNT语句

      SELECT COUNT(*) FROM students;
#查询students表有多少记录
#可以配合WHERE条件语句查询
#如果没有匹配到任何行，会返回0
    6. 多表查询

      大部分时候我们需要同时查询多个表的数据。这里就需要用到多表查询语句

      SELECT * FROM <表1> <表2>

      多表查询会把两个表的字段拼接到一起构成一张新的表返回。

      你应该想到，在两张表有相同的字段时，会让人分不清哪个是哪个。比如两张表都有id字段，多表查询会出现相同的id字段。所以我们可以使用 投影查询中的设置列别名来解决。

      SELECT
 students.id sid,
 students.name,
 students.gender,
 students.score,
 classes.id cid,
 classes.name cname
FROM students, classes;
#将classes的name重命名为cname，其它同理
    7. 连接查询

      在关系型数据库中往往需要利用其它表的数据来对主表进行查询。假设我们需要在结果集中同时包含其它表的列，那么可以使用INNER JOIN来实现

      SELECT s.id, s.name, s.class_id, c.name class_name, s.gender, s.score
FROM students s
INNER JOIN classes c
ON s.class_id = c.id;

      INNER JOIN<表名>条件语句（内连接语句）

      将需要额外链接的表加入查询脚本，选出两张表中都符合条件语句的记录

      ON 语句

      添加连接的条件,这里表示student表的class_id列与classes表的id列相同的记录进行连接。

      使用这两个语句，输出的时候classes表中的数据就会对应到students表上的数据。关系型数据库的查询就这么实现了。

      OUTER JOIN（外连接语句）

      1. LEFT OUTER JOIN 语句：
      SELECT s.id, s.name, s.class_id, c.name class_name, s.gender, s.score
FROM students s
LEFT OUTER JOIN classes c
ON s.class_id = c.id;

      这里会检索出左表（子表）中的全部记录，而右表（主表）只会输出符合条件的记录

      1. LEFT OUTER JOIN 语句：
      SELECT s.id, s.name, s.class_id, c.name class_name, s.gender, s.score
FROM students s
RIGHT OUTER JOIN classes c
ON s.class_id = c.id;

      这里会检索出右表（主表）中的全部记录，而左表（子表）只会输出符合条件的记录

      1. FULL OUTER JOIN 语句：

        检索出左表和右表全部的记录。
