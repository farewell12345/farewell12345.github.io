高并发下的单例模式

发表于 2020-09-12- 更新于 2020-09-13
643- 2m-
Java单例模式

什么是单例模式？

单例模式是为 确保一个类只能有一个实例，并为整个系统提供一个全局访问点的一种模式方法。

比如在一个大型项目中，我们需要设计一些配置文件，并把配置文件中的数据读到程序中。而为了便于随时调用这些配置，我们就需要把他们封装起来，比如封装一个Config类。而且这个类 只能有一个实例对象，因为在一个项目中不需要两套相同的配置。这时候就需要用到单例模式。

特点

  1. 在任何情况下，单例模式只有一个实例存在。

  2. 单例需要有能力为整个系统提供这唯一实例。也就是需要在任何地方都能获取到这个实例。

实现方式

  1. 饿汉式单例（提前创建）

  1. 惰性单例（什么时候用什么时候加载）

    
public class MySingleton {

 private static MySingleton instance = null;

 private MySingleton(){}

 public static MySingleton getInstance() {
     if(instance == null){//懒汉式
         instance = new MySingleton();
     }
     return instance;
 }
}

    但是惰性加载有一个问题，当处于高并发状态下时无法保证线程安全，会导致多个单例对象的出现，因此我们需要解决这个问题。

    高并发下线程安全的单例模式

    1. 在方法中使用线程锁(加synchronized关键字)，效率偏低

      
public class MySingleton {

private static MySingleton instance = null;

private MySingleton(){}

public synchronized static MySingleton getInstance() {
  try { 
      if(instance != null){//惰性加载

      }else{
          instance = new MySingleton();
      }
  } catch (InterruptedException e) { 
      e.printStackTrace();
  }
  return instance;
}
}

    2. 双检查锁懒加载（同步代码块的二次检查）

      class MySingleton {
//使用volatile关键字保其可见性
volatile private static MySingleton instance = null;

private MySingleton(){}

public static MySingleton getInstance() {
  if(instance == null) {//懒惰式
      //对重要代码进行单独同步
      synchronized (MySingleton.class) {
          if (instance == null) {//二次检查
              instance = new MySingleton();
          }
      }
  }
  return instance;
}
}

    3. 使用静态内部类实现单例模式（预加载）

      public class MySingleton {
//内部类
private static class MySingletonHandler{
  private static MySingleton instance = new MySingleton();
} 

private MySingleton(){}

public static MySingleton getInstance() { 
  return MySingletonHandler.instance;
}
}

    4. 使用static代码块实现单例(预加载)

      public class MySingleton{

private static MySingleton instance = null;

private MySingleton(){}

static{
  instance = new MySingleton();
}

public static MySingleton getInstance() { 
  return instance;
} 
}
