    最小路径和

    Posted on 2020-07-15- Edited on 2020-08-15
    858- 3m-
    leetcodec++

    题目描述

    给定一个包含非负整数的 m x n 网格，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

说明：每次只能向下或者向右移动一步。

示例:

输入:
[
  [1,3,1],
  [1,5,1],
  [4,2,1]
]
输出: 7
解释: 因为路径 1→3→1→1→1 的总和最小。
    来源：力扣（LeetCode） 链接：https://leetcode-cn.com/problems/minimum-path-sum **解法 一 暴力递归求解：** > 思路比较简单，也最容易想到，就是在每个节点处对向左和向右走选择权重比较低的那个进行运算，最后自底返回得出最短路径 > > 但是时间复杂度指数级别上升，因而这种**暴力算法不可取** > 
    class Solution {
public:
    int PathChoose(vector<vector<int>> grid,int row,int line)
    {
        if (row>=grid.size()-1&&line>=grid[0].size()-1)
        {
            return grid[row][line];
        }
        if (row==grid.size()-1)//如果到最右边就向下走
        {
            return grid[row][line]+PathChoose(grid,row,line+1);
        }
        if (line==grid[0].size()-1)//如果到底就向左走
        {
            return grid[row][line]+ PathChoose(grid,row+1,line);
            /* code */
        }
        return grid[row][line]+min(PathChoose(grid,row+1,line),PathChoose(grid,row,line+1));
    }
    int minPathSum(vector<vector<int>>& grid) {
        return PathChoose(grid,0,0);//直接超时，请看解法二
    }
};

    解法二 使用动态规划求解：

    在开始之前需要先了解一下什么是 动态规划(知乎大佬的文章，讲得很清楚)

    思路：

    动态规划问题最重要的就是构建 状态转移方程，所以首先针对题意构建状态转移方程。

    通过观察，可以注意到，到达节点grid[ i ] [ j ]的最短路径之和为节点 grid[i] [j]与位于其上一个节点之和

    比如矩阵

    [
  [1,3,1],
  [1,5,1],
  [4,2,1]
]

    到达[0] [1]也就是数字3处的最短路径为 1+3=4也就是 [0] [1]+[0] [0]

    到达[1] [1]也就是数字5处的最短路径为 5+1+1=7也就是 [1] [1] +（[1] [0] + [0] [0]）也就是由到[1] [0]点的最短路径+[1] [1]的路径长

    由此我们便可得出规律：到点[i] [j]的最短路径长为

    遍历全局之后终点的数值也就是到终点最短的路径和了

    上代码

    class Solution {
public:
    int minPathSum(vector<vector<int>>& grid) {
        int line=grid.size()-1;//行
        int row=grid[0].size()-1;//列
        for (int i = 0; i <= line; i++)
        {
            for (int j = 0; j<= row; j++)
            {
                if (i+j<=0)continue;//如果处于起点就跳过
                if (i==0)
                {
                    grid[i][j]=grid[i][j]+ grid[i][j-1];
                    //处于左边界时到达这里最短路径为上面的元素权重与这个的和
                }else if (j==0)
                {
                    grid[i][j]=grid[i][j]+grid[i-1][j];
                    //处于上边界时到达这里的最短路径为左面的元素权重与这个的和
                }else{
                    grid[i][j]=grid[i][j]+min(grid[i][j-1],grid[i-1][j]);
                    //离开边界的最短路径为上面和左面元素最小值和这个的和，也就是上个节点与这个的和
                }

            }
        }
        return grid[line][row];
    }
};
