    树

    线性表，栈，队列都属于线性结构。而树中的元素之间并不存在天然的直接后继或者直接前驱关系（说人话就是树具有一对多的关系，图具有多对多的关系，线性结构是一对一的关系）。但是只要附加某种约束（比如遍历），也可以在树中的元素之间确定某种次序，因此树属于 半线性结构

    树等价于连通无环图，因此与图相同，树也由一组 顶点（也叫结点或节点）和联结与其间若干条 边组成。

    有时我们还会在此基础上，指定某一特定顶点，并称之为 根（root)。在指定根节点之后，我们也将这棵树称为有根树。

    在上图中，A节点就是树的根，而K，L，F，G，M，I，J是这颗树的叶节点，叶节点没有任何子节点。

    树的节点：

    ​ 父节点：以上图中的树为例，对于图中的节点ABCD来说，A在BCD的上一层，是BCD的父节点（也叫双亲结点），BCD是A的子节点（也叫孩子结点）。

    ​ 兄弟节点：上图中BCD，同属一层中，所以为兄弟节点

    子树和空数树

    子树： 还是拿上图为例，如果单独将这颗树中的B,E,F,K,L节点拿出来，可以看到它们也可以组成一颗以节点B为根的树。所以称以B,E,F,K,L节点组成的树为 原树的子树

    由此我们可以得出推论： 树是由根节点和若干子树组成的

    空树：如果集合本身为空，那么由空集合构成的树就叫空树 空树没有节点

    树的度

    对于一个结点，拥有的子树数（结点有多少分支）称为结点的度（Degree） ，还是拿上图中的A节点为例，根节点A下有3个子树，因此A节点的度为3

    树的深度（层次）：

    由于树的连通性，每一个结点与根之间都有一条路径相连；而根据树的无环性，由根通往每个节点的路径必然唯一。由此可定义沿每个结点v到r的唯一通路所经过变得数目，称作v的深度（depth），记作depth（v）。依据深度排序，可以对所有结点做分层归类。

    特别地，约定根节点深度为0，属于第0层

    树的高度：

    对于某颗树中所有结点深度的最大值叫做树的高度

    有序树和无序树

    有序树

    树中任意节点的子结点之间有顺序关系，这种树称为有序树

    在有序树中，一个结点最左边的子树称为”第一个孩子”，最右边的称为”最后一个孩子”。

    无序树

    树中任意节点的子结点之间没有顺序关系，这种树称为无序树,也称为自由树

    森林

    由n(n>=0)个 互不相交的树组成的集合称为森林。还是拿上面图中的树为例，分别以B,C,D为根节点的三颗子树就可以称为僧侣。

    二叉树

    满二叉树:

    所有分支节点都存在左子树和右子树，并且所有叶子结点都在同一层上，这就是满二叉树。如下图

    二叉树中的度：二叉树中链接节点与节点的线就是 度，有n个节点，就有n-1个度，节点数总是比度要多一个。

    满二叉树特点：

    1.叶子只能出现在最下面一层

    2.非叶子节点度一定是2

    3.在同样深度的二叉树中，满二叉树的结点个数最多，叶子数量最多。

    完全二叉树：

    完全二叉树从根结点到倒数第二层满足完美二叉树，最后一层可以不完全填充，其叶子结点都靠左对齐 。

    特点： 叶子结点只能出现在最下层和次下层，且最下层的叶子结点集中在树的左部。

    需要注意的是，满二叉树肯定是完全二叉树，而完全二叉树不一定是满二叉树。

    如图：就是一颗完全二叉树

    一般地，树中个节点的孩子数目并不确定。每个节点的孩子均不超过k个的有根树，称为k叉树。

    二叉树的遍历

    前序遍历

    前序遍历的思路很简单，就是从左到右，从上到下依次遍历节点。

    拿上面的那个 三叉链表结构图来举例说明，前序遍历的顺序是 1->2->4->3

    也就是遍历完左子树后再返回对右子树进行遍历

    前序遍历有两种实现方式， 递归实现和非递归实现

    先来看最简单的递归实现

    //前序遍历(递归实现)
void PreOrderTraverse(Node *T)
{
    if (T != NULL && T->data != -1)//判定是否到底
    {
        cout << T->data << " ";//输出数据
        PreOrderTraverse(T->left);//遍历左子树，遍历到底返回
        PreOrderTraverse(T->right);//遍历右子树，遍历到底返回
    }
}

    然后是 非递归实现

    //前序遍历（非递归实现）
/*思路：依据递归思想，我们使用栈来模拟递归实现前序遍历
        首先将左子树一次入栈，直到遍历到左子树的叶节点（也就是遍历到没有左子树的节点为止)
        然后弹出栈顶元素，根据栈顶元素找到右子树，然后重复上面的操作，
        直到栈中元素全部弹出，且没有节点可进行入栈操作
*/
void PreOrder(Node *H)
{
       Node *T = H;
       stack<Node *> ptr;
       while (T != NULL || !ptr.empty())
       {
           //讲左子树入栈，直到遍历到左子树树叶上
           while (T != NULL)
           {
               if (T->data != -1)
               {
                   cout << T->data << " ";
               }
               ptr.push(T);
               T = T->left;
           }
           //遍历完左子树后跳出遍历右子树
           if (!ptr.empty())
           {
               T = ptr.top();
               ptr.pop();
               T = T->right;
           }
       }
}

    中序遍历

    在之前的学习中，我们得知前序遍历是由 根节点到孩子节点，自上而下，自左到右的遍历方式。

    而中序遍历是 由孩子节点到根节点，自下而上，自左到右的遍历方式

    以下图为例：

    从根节点1开始遍历根节点1的左子树，直到找到节点4。

    节点4无左孩子，也无右子树，因此节点2左子树遍历完成，遍历节点2的右子树

    节点4无左孩子，也无右子树，因此节点2右子树遍历完成，返回遍历节点1的右子树。

    重复步骤，直到遍历完成

    中序遍历输出顺序：

    而如果是前序遍历的话，输出顺序为：

    比较可以发现，中序遍历的根节点1在中间，而前序遍历的根节点1在开头

    中序遍历代码实现

    与前序遍历类使，中序遍历也可由递归实现和非递归实现

    /*中序遍历递归实现，没什么好说的*/
void INOrderTraverse(Node *T)
{
    if (T)
    {
        INOrderTraverse(T->left);//先遍历左孩子
        if (T->data!=-1)//输出语句在中间，而前序遍历在最前面
        {
            cout<<T->data<<" ";
        }
        INOrderTraverse(T->right);
    }
    return;
}

    中序遍历非递归实现

    /*与前序遍历的非递归实现方法类似，同样是使用栈来模拟递归,
但是前序遍历的非递归实现我们使用先输出再入栈的方式
而这里我们用的是先入栈再输出的方式，
这样就能确保遍历完全部左子树再进行输出*/
void InOrderTraverse(Node *T)
{
    stack<Node *> *p = new stack<Node *>;
    Node *Temp = T;
    while (Temp != NULL || !p->empty())
    {
        while (Temp != NULL)//先遍历到左孩子
        {
            if (Temp->data != -1)
            {
                p->push(Temp);
            }
            Temp = Temp->left;
        }
        if (!p->empty())
        {
            Temp = p->top();
            p->pop();
            cout << Temp->data << " ";//输出左孩子
            Temp = Temp->right;//转向遍历右子树
        }
    }
}

    后序遍历

    有了前序遍历和中序遍历的经验，后序遍历就显得比较简单了

    二叉树后序遍历的实现思想是：从根节点出发，依次遍历各节点的左右子树，直到当前节点左右子树遍历完成后，才访问该节点元素。

    还是来以这张图为例

    后序遍历的顺序是：

    • 从根节点1开始遍历左子树

    • 遍历根节点2的左子树

    • 遍历到根节点2的左孩子4并访问其中的数据

    • 从孩子节点4返回节点2，遍历节点2的右孩子节点5

    • 访问节点2的数据

    • 返回根节点1，开始遍历根节点1的右子树，方式与左子树相同

    • …

    • 遍历右子树结束后返回根节点1，并访问根节点1的数据

    • 输出：4 5 2 6 7 3 1

      可以发现，后序遍历时根节点最后访问

    递归实现：

    //后序遍历（递归实现）
void PostOrderTraverse(Node *T)
{
    if (T)
    {
        PostOrderTraverse(T->left); //先遍历左孩子
        PostOrderTraverse(T->right);
        if (T->data != -1) //后序遍历将输出语句移动到后面即可
        {
            cout << T->data << " ";
        }
    }
    return;
}

    非递归实现：

    二叉树后序遍历的非递归实现是三种遍历方式中最难的一种

    具体思路如下：

    1. 当前结点左子树不为空且左孩子和右孩子没有被访问过的情况下，不断入栈左子树；
    2. 左子树访问结束，判断当前结点是否有右孩子且右孩子没有访问过的情况下，入栈右孩子，回到步骤1；
    3. 当最后一个右孩子被访问到时，开始出栈，并记录上一次 范围的结点。
    void INPostOrderTraverse(Node *T)
{
    stack<Node *> p;
    Node *Temp = T;
    Node *r=NULL;//用于标记节点是否已经被访问
    while (Temp || !p.empty())
    {
        if (Temp )//遍历到最左子树最低端
        {
            p.push(Temp);
            Temp=Temp->left;
        }else//表示左子树访问结束
        {
            Temp=p.top();//取出栈顶元素，也就是节点
            if (Temp->right &&Temp->right!=r)//右子树存在且没有访问过
            {
                Temp=Temp->right;//转向遍历右边子树
            }else//表示结束了对该的遍历，进行访问并返回出栈的节点
            {
                p.pop();
                if (Temp->data!=-1)
                {
                    cout << Temp->data << " ";
                }
                r=Temp;//标记为最近访问，下次不对其进行访问
                Temp=NULL;
            }
        }
    }
}

    层序遍历

    故名思意，按照二叉树中的层次从左到右依次遍历每层中的结点，也就是逐层遍历二叉树。

    实现思路：

    通过使用队列的数据结构，从树的根结点开始，依次将其左孩子和右孩子入队。而后每次队列中一个结点出队，都将其左孩子和右孩子入队，直到树中所有结点都出队，出队结点的先后顺序就是层次遍历的最终结果。

    代码实现：

    void SequenceTraversal(Node *T)
{
    queue<Node *>p;
    p.push(T);//先将根节点入队列，表示第一层
    Node *Temp=T;
    while (Temp || !p.empty())
    {
        Temp=p.front();//弹出节点，将节点的左子树和右子树根节点入队列
        p.pop();
        cout<<Temp->data<<"  ";
        if (Temp->right->data!=-1)//判断左节点是否存在
        {
            p.push(Temp->left);
        }
        if (Temp->left->data!=-1)//判断右节点是否存在
        {
            p.push(Temp->right);
        }
    }
}

    线索二叉树

    现有一棵结点数目为n的二叉树，采用二叉链表的形式存储。对于每个结点均有指向左右孩子的两个指针域，而结点为n的二叉树一共有n-1条有效分支路径。那么，则二叉链表中存在2n-(n-1)=n+1个空指针域。那么，这些空指针造成了空间浪费。

    如图所示，在整个二叉树构建中一共有10个节点，有11个空指针，而如果使用中序遍历时遍历结果为HDIBJEAFCG 可以得知A的前驱结点为E，后继结点为F。所以我们可以将空闲的指针利用起来，将空闲的指针指向前驱节点或者后继节点，提升遍历速度。因此我们来定义规则如下：

    若结点的左子树为空，则该结点的左孩子指针指向其前驱结点。

    若结点的右子树为空，则该结点的右孩子指针指向其后继结点。

    排序二叉树的增删改查

    理解了二叉树的遍历以后我们来看看利用二叉树进行排序以及二叉树的增删改查

    //对树进行插入操作
void Tree::Insert_BST(Node *root, int data)
{
    Node *p = new Node;
    p->setData(data);
    if (root == NULL) 
        //如果根节点为空，也就代表整棵树为空树，直接赋值给根节点
    {
        root = p;
        return;
    }
    if (this->Search_BST(root, data) != NULL)
    {
        return; //查找要插入的数据是否已经存在于节点中
    }
    Node *T = NULL, *troot = root;
    while (troot)
    {
        T = troot;
        troot = (data < troot->getData()) ? troot->LeftChild : troot->rightChild; 
        //找到要插入的孩子节点，因为是排序二叉树，所以在插入时只需判断节点所含数据与要插入数据的的大小即可
    }
    if (data < T->getData()) //已经找到要插入的节点T,进行判断，如果数据比T大，则插入为左孩子，反之右孩子
    {
        T->LeftChild = p;
    }
    else
    {
        T->rightChild = p;
    }
    p->Parent = T;//为了方遍我们直接建立三叉链表，更方遍搜索查找
}
    //排序二叉树搜索操作
Node *Tree::Search_BST(Node *root, int data)
{
    Node *p = root;
    while (p)
    {
        if (p->getData() == data)
            return p;
        p = data < p->getData() ? p->LeftChild : p->rightChild;
    }
    return NULL;
}

    我们来主要说一下二叉树节点的删除操作

    因为二叉树的特殊性，我们在进行删除操作时要考虑三种情况：

    1.被删除节点同时存在左子树和右子树

    2.被删除节点只存在左子树或只存在右子树

    3.被删除节点没有孩子节点

    第2和第3很好理解，我们主要来说说第1点如何实现：

    考虑到删除以后遍历的顺序不能乱，所以不能直接删，必须考虑在一个树中重新找到一个合适的节点来替换这个被删除的节点。

    经过资料查询，得知这个节点默认是 从被删除节点出发，经过它的右节点，然后右节点最左边的叶子节点就是我们要找的替换节点。叫做 中序后继节点 故名思意，也就是中序遍历中该节点的后继节点

    我们来用图说明一下寻找中序后继节点并进行替换删除的流程：

    上代码：

    Node *Tree::getSuccessor(Node *p)//获得中序节点的下一个节点，传入要删除的节点
{
    Node *curr = p->rightChild;//拿到目标结点的右节点
    Node *successor = curr;
    Node *sucorPar = NULL;
    while (curr)
    {
        sucorPar = successor;
        successor = curr;
        curr = curr->LeftChild;
        //寻找右节点的最底部左孩子，也就是目标中序后继节点
    }
    if (successor != p->rightChild)
    {
        //将后继节点的子节点（仅仅可能有右节点）替补到后继节点的位置上
        sucorPar->LeftChild = successor->rightChild;
        //将被删除的右孩子连接到后继节点的右节点上
        sucorPar->rightChild = p->rightChild;
        //将被删除的左孩子连接到后继节点的右节点上（就是用后继节点替换掉被删除的节点）
    }
    return sucorPar;
}
    void Tree::Delete_BST(Node *root, int data)
{
    //搜索得到该被删除节点，并存储其父节点
    Node *p = Search_BST(root, data);
    Node *Parent = p->Parent;

    /*被删除节点即有左孩子，也有右孩子的情况*/
    if (!p->LeftChild && !p->rightChild)
    {
        Node *T = getSuccessor(p);
        //找到中序后继节点
        if (p==root)//替换
        {
            root=T;
        }
        delete (p);//删除
        return;
    }
    //如果p是父节点的左孩子
    if (Parent->LeftChild == p)
    {
        //将被删除节点的孩子或者子树链接到父节点
        if (p->LeftChild)//p只存在左孩子
        {
            Parent->LeftChild = p->LeftChild;
        }
        else if (p->rightChild)//p只存在右孩子
        {
            Parent->LeftChild = p->rightChild;
        }else{
            Parent->LeftChild=NULL;//既不存在左孩子也不存在右孩子
        }
    }
    //如果p是父节点的右孩子，其它同上
    else if (Parent->rightChild == p)
    {
        if (p->LeftChild)
        {
            Parent->rightChild = p->LeftChild;
        }
        else if (p->rightChild)
        {
            Parent->rightChild = p->rightChild;
            /* code */
        }else {
            Parent->rightChild=NULL;
        }
        /* code */
    }

}
