协程

也叫微线程，但需要注意的是，协程与多线程不同，协程是建立在单个线程上的.

协程也叫微线程，但需要注意的是，协程与多线程不同，协程是建立在单个线程上的.

特点：协程是由程序自身所管理控制，而非操作系统内核所控制。

下面这张图很好地说明了协程与线程的关系

解释：如果有一定编程基础，那么应该知道，多线程是靠操作系统内核进行管理，通过切片的方式来切换线程间的执行。在多线程执行时，往往会涉及到同步锁，线程阻塞状态、可运行状态之间的切换、线程上下文的切换，不管以上哪一点，都是非常耗时的操作，因此我们将单一线程拆分为多个协程，并发工作，省去了多线程切换的性能和时间损耗，大幅提升高并发性能

缺点：本质上还是单线程，无法利用单个CPU的多个核心，但是可以通过多线程嵌套协程的方式解决这个问题

总结：

  • 一个进程包含多个线程，一个线程可以包含多个协程，协程基于单个线程
  • 协程是跑在线程上的，一个线程可以同时跑多个协程
  • 协程是在程序层面进行任务切换的，而多线程是在操作系统内核层面进行切换的
  • 协程可以提高并发性能
  • 可以人为控制协程的挂起与运行

下面来学习如何在Kotlin中使用协程进行并发操作

  • 创建协程——coroutines

    首先导入

    import kotlinx.coroutines.*

    然后使用GlobalScope类中的launch函数创建一个协程

    GlobalScope.launch(Dispatchers.Unconfined) {
   print("World!")
}

    这里我们来详细看一下launch函数

    launch函数有3个参数和一个返回值

    我们先来看参数：

    • ​ 第一个参数是协程的上下文——CoroutineContext:

      ​ 我们可以设置CoroutineDispatcher 协程运行的线程调度器，可以指定协程运行在哪个线程，有 4种线程模式：

      1. Dispatchers.Default——默认模式，可有可无
      2. Dispatchers.IO -指定协程运行在共享的线程池中
      3. Dispatchers.Main - 主线程
      4. Dispatchers.Unconfined - 没指定，就是在当前线程

      我们可以自己创建协程的上下文，也就是 线程池

      val signgle= newSingleThreadContext("aa")//创建单线程
GlobalScope.launch(signgle) {
    delay(500L)
    print("World!")
}
val signgles= newFixedThreadPoolContext(2,"nn")//创建线程池
GlobalScope.launch(signgles) {
    Fun2()
    Fun1()
}

    • CoroutineStart——启动模式

      ​ 也就是选择创建后就立即启动协程还是懒惰等待，等调用的时候再启动

      模式：

      1.DEAFAULT -默认，创建就启动

      2.LAZY，懒惰加载，也就是等到调用的时候再启动

      来看个lazy启动的示例:

      val job:Job=GlobalScope.launch(start = CoroutineStart.LAZY) {
    print("World!")
}
print("hello")
job.start()//手动启动协程
Thread.sleep(1000L)//让主线程等待协程结束

    • block - 闭包方法体，定义协程内需要执行的操作

    • Job - 协程构建函数的返回值，可以把 Job 看成协程对象本身，协程的操作方法都在 Job 身上了

      Job对象的相关函数：

      job.start ( ) - 启动协程，除lazy模式，协程都不需要手动启动

      job.join ( ) - 等待协程执行完毕，注意：join函数只能在协程中使用

      job.cancel ( ) - 取消这个协程

      job.cancelAndJoin ( ) - 等待协程执行完毕然后再取消

  • 带返回值的协程——GlobalScope.async

async与launch的区别是async是有返回值的，创建新的协程，默认运行在后台新的线程中，并且不堵塞当前线程，支持通过await获取返回值

注：async需要运行在launch协程或者runBlocking中，

async 返回的是 Deferred 类型，Deferred 继承自 Job 接口，Job有的它都有，增加了一个方法 await ，这个方法接收的是 async 闭包中返回的值，async 的特点是不会阻塞当前线程，但会阻塞所在协程，也就是挂起

runBlocking {//堵塞当前线程以确保协程运行结束，下文会详细讲
        val deferred = GlobalScope.async {
            return@async "I am async End!"//通过标签的形式返回值
        }
        print(deferred.await())//采用await进行输出返回值
    }

  • runBlocking——阻塞当前线程的协程

    ​ 在进行launch的时候我们需要用sleep来阻塞主线程来等待协程结束，这样无疑会增加一下不必要的工作，而runBlocking可以阻塞当前线程，让线程等待协程运行结束后才会继续运行。这就类似在协程之后加了一个 Thread.sleep()

    runBlocking {
      print("I am runBlocking")
}
print("over")
