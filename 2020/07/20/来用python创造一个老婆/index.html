来用python创造一个女朋友

发表于 2020-07-20- 更新于 2020-08-15
4.8k- 22m-
python机器人插件

常会听到 一些单身狗程序员（比如我）调侃自己可以“new一个对象”，那么今天我们来把这个调侃真正地实现一下，用代码创造一个可以与你网恋的“虚拟女友”。

先来看看目前我初步开发的效果（还有很多不足，个人也比较菜，求大神轻喷）：

准备工作

首先是平台，毕竟“虚拟女友”是不可能有实体的，因此我们需要选择适当的社交平台，以机器人为媒介创造我们的“虚拟女友”，主流社交平台无非就两个，微信和QQ，在这里我选择了更为成熟的QQ机器人框架——基于cqhttp的NoneBot框架，开发环境为python3.7.

思路构建

形象构建

首先我们拥有一个女朋友对象，她有外貌特征，性格特征，对我们的好感度，姓名等属性，因此我们在new一个女朋友时要把这些全部考虑在内。为此我手动在网路上搜集了一百余个中国常见姓氏和一百余个女性常用名（大多取自诗经），我在这里直接贴在下面，如果有需要直接复制或者去此项目的github仓库中fork文件即可。

然后就是选取我们已经准备好的数据来构建一个完美的女朋友。在这里可以直接使用随机数来抽取姓氏和名字等属性元素进行组合。如果您是大佬，可以考虑一下根据生成的性格等因素来给外貌等属性赋予权值，达到更加细致的形象刻画。

注：姓氏和名字我已贴出，性格，外貌等属性可以依据个人喜好进行编辑，生成方式与姓氏同理

性格构建

女朋友的性格决定了她对你好感的增长快慢和每日与你聊天时的话语，因此女朋友性格对于之后的相处很重要。最简单的方法就是根据不同的性格构建不同的语料库，然后使用ChatterBot进行训练（非必要，大佬可以左转NLP手撸，萌新可以直接写点语录或者基础对话，进行对话时随机发送语料）或者如果你很懒的话，可以直接调取图灵API

事件构建

处于恋爱中的男女大都会每日早晚安打卡，不定时与伴侣分享自己的所见所闻所感，因此我们可以编写一个事件语料库，不定时触发语料库中的事件，向你进行诉说，或者每日向伴侣说“早安晚安”

爱称构建

部分处于恋爱中的情侣互相之间都有爱称，因此我们也可以给女朋友对象赋予一个“昵称”属性，用来亲切地呼唤她，也可以设置专有爱称，让她以爱称来呼唤你。

数据存储构建

有了女朋友，我们就需要想办法留住她，给她一个静态的数据文件用来保存，目前我认为最好的方式是使用json保存女朋友的数据，方便存取。
