缺失数字

Posted on 2020-07-14- Edited on 2020-08-15
865- 3m-
leetcodec++

题目描述：

给定一个包含 0, 1, 2, …, n 中 n 个数的序列，找出 0 .. n 中没有出现在序列中的那个数。

示例 1:

输入: [3,0,1] 输出: 2

示例 2:

输入: [9,6,4,2,3,5,7,0,1] 输出: 8

来源：力扣（LeetCode）

链接：https://leetcode-cn.com/problems/missing-number

思路构建

首先题目给出的数集内的元素都是正整数，而且在0…n的范围内只缺少一个数，

那么也就是说除这个数外其它数都是连续的

再进一步思考，如果直接读取这些数并将这些数的数值分别作为索引去访问并将对应的索引元素进行标记。

最后进行统计，没有被标记的索引元素就是这串序列缺失的数

比如数集A=[5,4,3,2,0]，我们先读取第一个元素的数值“5”,然后去访问A[5]并进行标记。

再读取第二个元素“4”,然后去访问A[4]并标记…以此类推。

最后我们会发现A[1]这个索引没有被访问过，因此也就没有被标记，便可以很快速的找到这个缺失的数字“1”

作为聪明读者也许你已经发现了上面的思路有一个重大的bug，那就是在访问A[5]的时候会发生数组越界的异常，因此我们需要在进行这些操作之前给数集补一个空位，避免这个异常的产生。

代码实现

#include<iostream>
#include <algorithm>
#include<vector>
using namespace std;
class Solution {
public:
    int missingNumber(vector<int>  &nums) {
        const int n =nums.size();//取输入的数集的大小
        nums.push_back(-2);
        /*在数集最后补一位用于填充
          在直接对数字进行搜寻时可能会出现数组越界的error。
          出现此error的原因是在输入的数集中存在与数集大小相等的数，导致在直接取对应位置的数据时产生越界异常
          因此在此补足最后一位防止产生越界问题
          至于为什么是-2，原因是作为空位不能出现对程序运行有干扰的数，因此不能出现正整数，而-1是我们的标记数，因此选-2，如果你想选-3，-4，-5都可以
        */
        const vector<int> temp=nums;//对输入的数集进行复制
        int ns =temp[0];//对第一个数据进行预处理
        nums[ns]=-1;//尝试找到第一个数据在数集中对应的索引数据
        for (int i = 1; i <n; i++)//对数集剩余数据进行遍历，同样是根据数据大小取索引
        {
            ns=temp[i];
            nums[ns]=-1;//成功访问索引后将该索引对应的值进行标记梗概
        }
        //数据处理结束后进行统计，在原数集中没有被标记为-1的就是所缺失的数
        for (int i = 0; i < n; i++)
        {
            //如果这个索引在数集中对应的值不是-1并且存在于原数集中，那么就找到了这个数
            if (nums[i]==temp[i] && nums[i]^temp[i]!=1)
            {
                return i;
            }
        }
        return n;   //如果所有数据都被标记，则说明缺失了数集的最后一位   
    };
};
