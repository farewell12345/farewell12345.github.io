简单爬取百度百科词条

Posted on 2020-08-15- Edited on 2020-09-17
453- 2m-
python机器人插件

一时兴起想写个可以自动百度的QQ机器人，底层代码不多，只有十几行，开发环境为pycharm+python3.8

思路

  1. 百度百科的界面属于静态网页，因此爬取难度并不高
  2. 采用get方法，并在header里添加设备信息即可获取完整的搜索结果
  3. 使用BeautifuSoup库进行静态网页解析
import requests
from bs4 import BeautifulSoup
import html.parser

baidu_wiki = 'https://baike.baidu.com/item/'
#传入要查询的词条，返回词条简介
def get_knowledge(txt:str)->str:
    headers = {
    'user-agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) '
                  'AppleWebKit/537.36 (KHTML, like Gecko) '
                  'Chrome/81.0.4044.138 Safari/537.36 Edg/81.0.416.77 '
    }  # 请求header
    try:
        data=requests.get(baidu_wiki+txt,headers=headers).text
        bs=BeautifulSoup(data,'html.parser')\
           .find_all(class_='para')[0]\
           .get_text()
        return bs
    except :
        return "很抱歉没有找到相关结果"

底层逻辑写好了，接下来需要封装入QQ机器人中，让它在QQ群里发光发热

这里依然选用了NoneBot机器人框架

为了保证响应速度和效率，采取了多线程配套协程的处理方案，可以大大提升机器人处理消息命令的效率

import threading

import nonebot
import asyncio
from bs4 import BeautifulSoup
from nonebot import *
import requests
import html.parser
baidu_wiki = 'https://baike.baidu.com/item/'


def get_knowledge(txt: str,session:CommandSession):
    #在多线程中开启新协程
    loop=asyncio.new_event_loop()
    asyncio.set_event_loop(loop)
    headers = {
    'user-agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) '
                  'AppleWebKit/537.36 (KHTML, like Gecko) '
                  'Chrome/81.0.4044.138 Safari/537.36 Edg/81.0.416.77 '
    }  # 请求header
    try:
        data = requests.get(baidu_wiki + txt, headers=headers).text
        bs = BeautifulSoup(data, 'html.parser')\
            .find_all(class_='para')[0]\
            .get_text()
        #获得目标文章并发送到QQ
        loop.run_until_complete(session.send(bs))
    except:
        loop.run_until_complete(session.send("很抱歉没有找到相关结果"))


@on_command('baidu', aliases="百度", only_to_me=False)
async def baidu(session: CommandSession):
    #使用多线程处理命令
    threading.Thread(target=get_knowledge, 
                     args=(session.current_arg,session,)
                    ).start()
