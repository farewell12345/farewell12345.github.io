什么是JWT

Posted on 2020-08-02- Edited on 2020-08-07
1.2k- 4m-
后端开发web

在开始之前，我们需要先了解 什么是token以及它和cookie，session的区别：

在交互式web中经常会需要管理会话，也就是记住哪些人登录了系统，并在一段时间内保持用户的登录，但是我们在后端中还必须把每个人区分开，防止数据混杂。因此最简单的办法就是给每个用户一个独一无二的标识，让后端能够根据表示轻易区别开每个用户。

于是产生了session和cookie，类似于私人与服务器交互的密钥，服务器以此来区分每个用户。

cookie:

cookie存储的数据容量有限，一般不超过4KB， 第一次登录后服务器返回一些数据（cookie）给浏览器，然后浏览器保存在本地，当该用户发送第二次请求的时候，就会自动的把上次请求存储的cookie数据自动的携带给服务器，服务器通过浏览器携带的数据就能判断当前用户是哪个了，cookie几乎没有加密，因此很容易被窃取和伪造，安全性比较低。

session:

session在客户端和服务端都会存储，安全性更高一些，而且存储的信息更多，这样做有个缺点，那就是个人和服务器都需要保存这个session，个人当然无关紧要那点保存空间，但是对于服务器来说需要保存成千上万的session，会严重印象服务器的效率。

于是token诞生了，token让服务器不必单独保存验证密钥，同时很好地解决了session被伪造的问题。

token原理：

举个例子，比如用户A通过ID和密码登录了系统，登录时向后端发送了A的ID和密码，后端会根据用户A的ID来计算生成一段签名，把这个签名存在token中然后发送给A的客户端，客户端保存这个token，当A的客户端把token再次发给服务器的时候，会顺带把A的ID也发过去，然后服务端再次对这些数据进行计算，得出一个签名，将新计算出的签名与发送来的token中包含的签名进行比对，以此来得知，A已经登录过了。

理解了token，那么接下来JWT就很好理解了:

JWT全称（json web token），看词组就能明白它的意思。官方中文解释是JWT是一种 基于JSON的开放标准（(RFC 7519).该token被设计为紧凑且安全的，特别适用于分布式站点的单点登录 （web）。

说人话就是JWT是用于生成token的一套方案。

这里我们来看一个简单的JWT生成token和解析token的例子（项目地址）

注：下面的例子用到了Java的JWT库和阿里的fastjson库（因为fastjson支持map和json的快速转换，虽然我们都知道它很辣鸡）

把依赖丢一下：

implementation 'com.alibaba:fastjson:1.2.73'
implementation 'com.auth0:java-jwt:3.10.3'

先来用JWT把一个成绩的JSON数据变成一个有时效的token：

    public static final String SECRET="JKKLJOoasdlfj";
//token密钥签名,可以自己随便定，但是定好不要随便改
    public static final int calendarField= Calendar.DATE;//一个月中的第几天
    public static final int calendarInterval=10;//时效，token时效为10天
    /*创建一个token,传入json数据，将json数据封装在token里并返回该token*/
    public static String CreateToken(JSONObject Grades) throws Exception{
        Date iatDate=new Date();
        Calendar nowTime=Calendar.getInstance();//获取时间戳
        nowTime.add(calendarField,calendarInterval);//设置时间格式
        Date expiresDate=nowTime.getTime();        //token的header
        JWTCreator.Builder A= JWT.create().withHeader(Grades);
        /*遍历json，添加进token的Claim中*/
        for (String i:Grades.keySet()) {
            A.withClaim(i, String.valueOf(Grades.get(i)));
        }
        return A.withIssuedAt(iatDate)
                .withExpiresAt(expiresDate)
                .sign(Algorithm.HMAC256(SECRET));
        //分别设置token有效期和对已构建好的JWT对象进行HMAC256计算
    }

创建好token，我们还需要对他进行解析，好获取其中包含的信息

public static Map<String, Claim> veifyToken(String Token){
        DecodedJWT jwt=null;
        try{
            JWTVerifier verifier=JWT.require(Algorithm.HMAC256(SECRET)).build();
            jwt=verifier.verify(Token);
            //对token进行解包
        }catch (Exception e){
            e.printStackTrace();
        }
        assert jwt != null;
    //返回一个解析后的map
        return jwt.getClaims();
    }

这里返回的map是Claim对象，每个对象中封装了我们需要的数据，因此我们在对map进行取值的时候需要调用Claim对象的asString()等方法来获取我们所需要的数据，比如：

JSONObject A=new JSONObject();
A.put("数学","150");
String T=CreateToken(A);
veifyToken(T).get("数学").asString()
    //输出我们之前已经设置好的数学成绩：150
